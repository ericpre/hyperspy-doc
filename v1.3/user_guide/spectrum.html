Spectrum tools¶

These methods are only available for Signal object with signal_dimension equal to one.

Cropping¶

In addition to cropping using the powerful and compact Signal indexing syntax the following method is available to crop spectra using a GUI:

The crop_spectrum(), method is used to crop the spectral energy range. If no parameter is passed, a user interface appears in which to crop the spectrum.

Background removal¶

The remove_background() method provides background removal capabilities through both a CLI and a GUI. Current background type supported are power law, offset, polynomial and gaussian.

Calibration¶

The calibrate() method provides a user interface to calibrate the spectral axis.

Spectral alignment¶

The following methods use sub-pixel cross-correlation or user-provided shifts to align spectra. They support applying the same transformation to multiple files.

  • align1D()
  • shift1D()

Integration¶

The integrate_in_range() method provides a GUI and a CLI to integrate the 1D signal dimension in a given range using the Simpson’s rule.

Data smoothing¶

The following methods (that include user interfaces when no arguments are passed) can perform data smoothing with different algorithms:

  • smooth_lowess()
  • smooth_tv()
  • smooth_savitzky_golay()

Spikes removal¶

New in version 0.5.

spikes_removal_tool() provides an user interface to remove spikes from spectra.

Spikes removal tool.

Other methods¶

  • Apply a hanning taper to the spectra hanning_taper()
  • Find peaks in spectra find_peaks1D_ohaver()
  • Interpolate the spectra in between two positions interpolate_in_between()
  • Convolve the spectra with a gaussian gaussian_filter()
